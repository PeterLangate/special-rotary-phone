Chapter 1 — Introduction

Chapter 1Meet PointClaw

PointClaw is a PowerPoint add-in from kalmstrom.com Business Solutions. It opens a chat panel on the right side of your presentation window, and from that panel you can ask an AI to build, edit and refine your slides — using plain English instead of menus, dialogs and mouse clicks.

1.1 What it is

PointClaw is a Microsoft PowerPoint task-pane add-in built with VSTO (Visual Studio Tools for Office). It runs inside the PowerPoint process — not as a separate app — and drives your open presentation through the native PowerPoint object model. Every change it makes is a real PowerPoint edit that you can see happening in front of you, undo with Ctrl+Z, and save with the file in the normal way.

Under the hood it talks to Anthropic's Claude language model for conversation and reasoning, and to Google's Imagen and Veo models for image and video generation. You supply your own API keys — the add-in does not run through any kalmstrom-hosted service.

1.2 What makes it different

Lives inside PowerPoint

No separate app, no browser, no copy and paste. The add-in talks directly to the slide deck you already have open.

Works on real slides

Every action is a normal PowerPoint edit. You can undo, redo, tweak or throw away any result just like your own work.

One hundred specialist tools

PointClaw does not guess with HTML or screenshots. It ships more than one hundred purpose-built tools for slides, shapes, tables, charts, animations, themes, speaker notes, media and deck audits — roughly three times the surface area of any community PowerPoint MCP server, with full parity on every file-level operation they expose.

1.3.1 How it compares to community PowerPoint MCP servers

There is a growing ecosystem of open-source Model Context Protocol servers that expose PowerPoint operations to external AI clients — most prominently the python-pptx-based servers such as GongRzhe's Office-PowerPoint-MCP-Server. They are excellent at scripted file-level work — writing a fresh .pptx on disk, stamping templates, extracting text for a pipeline — but they do not run inside PowerPoint and therefore cannot touch any feature the file format alone does not express.

PointClaw ships 108 specialist tools. The most fully featured community PowerPoint MCP server at the time of writing exposes 34. PointClaw covers every capability that server offers, and adds more than seventy tools only an in-process add-in can reach — triggered animations, motion paths, SmartArt, Imagen/Veo media generation, deck audits, PDF and video export, accessibility checks, screen capture, live slide thumbnails, theme-level formatting across every shape type, chart-series animation, connector routing between named shapes, slide-master editing, template file inspection, deck-wide font analysis, and coordinated "apply professional design" theme composition.

Rule of thumb: a PowerPoint MCP server edits a file. PointClaw edits the live presentation you are looking at, with every change visible, undoable, and signed off by you.

1.4 Who it is for

  • Presenters and trainers who build decks regularly and want an assistant that can take the first pass at structure, text and visuals.
  • Consultants and analysts who need to turn notes, briefs and bullet points into a presentable deck.
  • Teachers and educators who build lesson slides and want to generate supporting illustrations without leaving PowerPoint.
  • Anyone who would rather describe what they want in words than build every element by hand.
Plain speech works. You do not need to learn command names. "Add a two-column slide titled Roadmap with a bulleted list on the left and an illustration on the right" is a perfectly good instruction.

Chapter 2Installation & Setup

Before you can use PointClaw you need to install the add-in, get two API keys, and enter them in the settings window. The whole process takes about five minutes.

2.1 Install the add-in

  1. Close every open PowerPoint window.
  2. Run the PointClaw installer (PointClaw Setup.exe or the ClickOnce link supplied by kalmstrom.com).
  3. Follow the wizard — accept the prompts to install the Visual Studio Tools for Office (VSTO) runtime if it asks.
  4. Open PowerPoint. On the Home tab you should see a new PointClaw button.
The button is missing? Go to File → Options → Add-ins. At the bottom, change Manage to COM Add-ins, click Go… and make sure VSTOPPTPointClaw is ticked.

3.3 How PointClaw thinks

Under the hood, every instruction you type goes to Claude together with a short description of your current presentation (file name, slide count, which slide is selected). Claude breaks the request into one or more tool calls — precise operations like add a slide at position 2 with the title-and-content layout — which PointClaw then executes against the live PowerPoint document. After each batch of edits, Claude sees the result and decides whether it is done or needs to do more.

You do not need to know which tool is called when. But it helps to know that:

  • PointClaw can see the structure of your slides (which placeholders and shapes exist, what text they contain) but not the visual appearance unless you explicitly ask for a thumbnail. If something looks wrong, describe it — "the title is running off the edge" — and it will investigate.
  • It works best when you give it clear endpoints: "add four slides with titles A, B, C, D" is easier than "keep adding slides until the deck feels right".
  • It will tell you what it did. Read the reply — if something was ambiguous, PointClaw says what it assumed.

3.4 Undo is your friend

Every edit PointClaw makes is a normal PowerPoint edit, so Ctrl+Z undoes it. If an instruction goes wrong, undo your way back and try rephrasing. Nothing is destructive unless you asked for it.

4.2 Duplicate, move and delete

These are exactly what you would expect, and PointClaw understands positional language.

  • Duplicate: "Duplicate slide 3." The copy is inserted immediately after the original.
  • Move: "Move slide 7 to position 2." Or: "Move the pricing slide to just before the summary."
  • Delete: "Delete slide 4." Multiple at once: "Delete slides 4, 5 and 6."
Deletion is not undoable by PointClaw. Only PowerPoint's Ctrl+Z can bring a deleted slide back. Use undo immediately if you change your mind.

4.3 Navigate to a slide

If the slide you want to work on is not on screen, PointClaw can switch to it: "switch to slide 12" or "go to the roadmap slide". This puts that slide in PowerPoint's normal view so you can see the changes as they happen.

4.4 Get the lay of the land

When you open an existing deck you did not build yourself, ask PointClaw to summarize what is there:

Give me an outline of every slide in this deck — title, key points, and any speaker notes.

The reply is a structured summary you can skim, followed by suggestions for where to edit.

Other read-only inspection commands

  • "What shapes are on slide 4?" — lists every shape, its name, position and size.
  • "Show me the content of slide 7." — full text of every placeholder and textbox, plus notes.
  • "Take a thumbnail of slide 2." — PointClaw exports a PNG so it can see the slide visually. Use this when you suspect a layout problem.

4.5 Create, open and describe a deck

PointClaw can also handle the file-level operations that used to require the File menu. These commands work from the chat pane directly:

  • Create a new presentation. "Create a new blank presentation." PowerPoint opens it in a new window next to your current one. Add a save path if you want it on disk immediately: "Create a new presentation at C:\Decks\Draft.pptx."
  • Start from a template. "Create a new presentation from C:\Templates\Brand.potx." PowerPoint opens the template as an untitled copy — the template file itself is not modified.
  • Open an existing file. "Open C:\Decks\LastQuarter.pptx." The file opens in its own PowerPoint window. Your chat pane continues to work on the deck it started in, so you can use this to reference or copy from another deck without losing your place.
  • Set document metadata. "Set the title to Q2 Roadmap, the author to Peter, and add the keyword 'planning'." These are the same fields you see in File → Info — title, author, subject, keywords, category, comments, manager and company. Any field you do not mention is left alone.

4.6 Templates, slide master and deck-wide design

Once you have content to work with, PointClaw can reach into the deck's underlying structure — the template it was built from, the slide master that controls every slide, and the theme that sets the overall look:

  • Inspect a template before using it. "What's in C:\Templates\Brand.potx?" calls get_template_file_info and reports every custom layout, its placeholder types, accent colors, and theme fonts. get_template_info gives a lighter summary.
  • Edit the Slide Master. "Add a deck-wide footer 'Q2 2026 — Internal' and turn slide numbers on." This writes to the one master that every normal slide inherits from.
  • Fill a slide from a structured template. "Set slide 4's title to 'Roadmap', subtitle to 'Q2 deliverables', and add these bullets as the body: …" — apply_structured_slide_template matches each field to the right placeholder type, with body lines turning into bullets.
  • Apply a coordinated design in one call. "Apply a professional design: accent1 #0078D4, accent2 #F0A030, heading font Segoe UI, body font Inter." This composes set_theme_colors and set_theme_font into one transaction.
  • Update chart data. "Change the data behind 'Chart 2' on slide 7 to these categories and values." PointClaw rewrites the chart's embedded Excel workbook and keeps your formatting.
  • Add a connector. "Draw an elbow connector from BoxA to BoxB on slide 3." Identifies shapes by name; the connector auto-routes and updates when you move either endpoint.
  • Optimize text fit. "Make the text on slide 9 fit the placeholders" — optimize_slide_text enables word-wrap, shrinks oversized text, and bumps anything smaller than 14 pt up to a readable size.
  • Audit fonts. "What fonts are used in this deck?" — analyze_fonts tallies every font across every slide and warns when a deck mixes more than two families.
  • Set document metadata. "Set the title, author and keywords for this deck." Same fields you see in File → Info.

Chapter 5Text, shapes and tables

Once the slide structure is in place, PointClaw can fill it with the content that makes it useful — titles, bullet points, call-out shapes, tables, and formatting. This chapter covers every text and shape tool available.

5.1 Writing into placeholders

Most of PowerPoint's layouts have placeholders — pre-positioned boxes for the title, body text, and so on. PointClaw fills these directly instead of dropping a new text box on top:

On slide 5, set the title to "Three reasons we win" and the body to three bullet points: "Faster time to market", "Lower unit cost", "Simpler operations".

Placeholders are identified by index (1 is the title, 2 is the body, etc.). You do not normally need to know the numbers — just say "title", "body", "left column", "right column".

5.2 Free text boxes

When you need text in a specific spot, ask for a text box. Sizes and positions are in PowerPoint points by default, but everyday language works too:

Add a small text box at the bottom-right corner of slide 1 that says "Confidential — Do not distribute", 10 pt, grey.

5.5 Shapes

PointClaw can add any auto-shape PowerPoint supports — rectangles, ovals, arrows, callouts, stars and so on. Give it a type, a position, a size and (optionally) a fill color and a label:

On slide 6, add three equally spaced rounded-rectangle shapes across the middle, each 4 cm wide, labeled "Discover", "Decide", "Deliver". Fill them with the kalmstrom blue, white text.

Reformatting existing shapes

Once a shape is on a slide, PointClaw can change its fill, transparency, border color, border thickness and visibility. It identifies shapes by name (PowerPoint auto-names them like Rectangle 3) but you can also say "the blue rectangle on slide 6" and it will look it up.

5.6 Tables

Tables are built in two steps: add the grid, then fill the cells.

  1. "On slide 4, add a 3-column, 5-row table centered in the lower half of the slide." — creates the empty grid.
  2. "Fill the header row with 'Region', 'Sales', 'Growth'. Fill the data rows with …" — populates each cell.

You can also say it all in one sentence — PointClaw will chain the calls — but for complex tables the two-step form gives you a chance to inspect the grid first.

Existing tables. PointClaw can only fill cells, not restructure tables after creation. If you need a different row or column count, delete the table and create a new one.

6.4 Generate a video with Veo

Veo creates short video clips (a few seconds each) from a prompt. The call takes two to five minutes to complete — PointClaw shows a progress status while it runs.

Patterns that make PointClaw more reliable and keep API costs in check — plus what to do when something goes wrong.

9.1 Getting better results

Be specific about positions and sizes

"Put the illustration on the right" is good; "put a 6 cm illustration in the right half of the slide, vertically centered" is better. PointClaw cannot see the slide visually unless you ask for a thumbnail, so specific numbers prevent guesswork.

Break big requests into stages

Long, compound instructions ("build a 20-slide deck with illustrations and speaker notes") run longer and cost more. Split the work: outline first, fill text, then add visuals, then add notes. You will also get a chance to course-correct at each stage.

Use the right Claude model for the job

Switch to Haiku for repetitive bulk tasks (writing 20 speaker notes in a consistent voice) and to Opus for anything that needs judgment (restructuring a deck's argument, rewriting a subtle title).

Commit to a template early

If you have a branded template, open a file based on it before you start asking PointClaw to build slides. That way every new slide uses the right master layouts, colors and fonts automatically.

Chapter 10Design principles and the deck audit

PointClaw ships with a built-in opinion on what makes a good slide. That opinion lives in two places: baked into the system prompt so Claude follows it while building and editing slides, and expressed as an automated audit you can run against any open deck. This chapter explains both.

10.1 Why opinionated design?

Most presentations fail the same way: wall-of-bullets slides, 12 pt body text, six competing fonts, a title that says "Update" instead of naming the point. The remedies are known — and they compress into a short list. PointClaw bakes that list into the AI so you get them applied by default, and into an audit tool so you can catch the ones that crept in anyway.

10.2 Principles baked into the system prompt

Every message you send to PointClaw is prefixed with a system prompt that tells Claude how to behave. The default prompt includes a Design principles block instructing the model to apply these rules proactively — not wait to be asked. Here is the short version of what Claude is told:

Content & story

  • One idea per slide. Split if more than 5 top-level bullets or 40 words of body.
  • Titles are full sentences with a verb ("Q3 revenue beat plan by 12%"), not labels ("Q3 Results"). At least 4 words.
  • Prefer a chart, diagram, or headline over a bullet list. Bullets are a last resort.
  • Put detail — numbers, sources, caveats — in speaker notes, never on the slide.
  • Structure: Context → Conflict → Resolution → Ask. Never end on "Thank you" or "Questions?".

Typography

  • Body ≥ 24 pt, titles ≥ 36 pt on 16:9 decks. Never below 18 pt — split the slide instead.
  • Max two fonts. Sans-serif for body (Segoe UI, Inter, Calibri, Aptos). Serifs are for print, not projectors.

Colour & theme

  • Always use the theme. Set brand colours once; fill shapes by referring to accent1…accent6, not hard-coded hex.
  • Palette: 1 primary + 1 accent + neutrals. Red/green reserved for negative/positive data.
  • Text-on-background contrast must clear 4.5:1 (WCAG AA).

Layout, charts, motion, accessibility

  • Whitespace is a feature. A centred single element often beats a busy slide.
  • Alignment is a promise — use align_shapes and distribute_shapes; never eyeball it.
  • Chart the data, don't table it. Strip gridlines, 3-D, shadows, default legends.
  • Animations: only appear, fade, wipe, fly, zoom. Never bounce, swivel, spin.
  • Transitions: None, Fade, or Morph. Never Vortex / Airplane / Origami / Cube.
  • Every picture needs alt text. Every slide needs a title. Reading order must match logical order.

10.3 Overriding the defaults for your house style

The design block is part of the system prompt, so you override it the same way you override any other prompt behaviour — in Settings → System prompt. Two patterns work well:

Append your house style

Leave the defaults in place and add a short block at the end describing where you differ. Example:

House style overrides: our corporate template uses Arial (not Segoe UI). Body is 20 pt, not 24 pt. Titles always end with a period. Accent1 is our red #E63946; reserve it for section headers, not data positives.

This keeps every default Claude already knows about (alt text, contrast, animation allowlist, story structure) and only changes the handful that differ for your brand.

Replace the block entirely

Advanced. If your organisation has a formal design system, you can delete the default design block and paste your own rules in its place. Make sure you keep enough guidance for Claude to make judgement calls on slides you haven't explicitly described — an empty design block means generic output.

Reset is safe. The Settings window has a Reset button that restores the shipped default prompt, design block and all. You can experiment without losing the baseline.

Error vs Warn vs Info. Errors are unambiguous breakage — unreadable text, missing alt text. Warns are strong smells that a human should confirm. Infos are hints for polish. The audit never changes your deck on its own; every fix is a follow-up tool call you approve.

Auto-fix for the mechanical ones

A sibling tool, audit_deck_fix, applies the fixes that don’t need judgement: transition consistency (R24), vertical-centring a text box inside its wrapper shape (R27), horizontally centring a sole hero shape (R28), and aligning two shapes on the same row (R30). Run it when you just want the mechanical things tidied up, then run audit_deck to see what remains:

Run audit_deck_fix on this deck, then audit_deck again and summarise what’s left.

Judgement calls — titles, alt-text wording, copy rewrites — are intentionally not auto-fixed. Those still need you (or a follow-up prompt that commits to a specific change).

Two safety guards worth knowing

  • Broad find-and-replace is blocked for short acronyms. Case-insensitive replace of a 2–4-letter all-caps string ("AI", "ML", "API") would damage embedded occurrences ("PointClaw", "APIs").   • Hide feedback names the slide. set_slide_hidden echoes back the slide title so you can verify the right slide was hidden before later edits shift indices.

Each line is actionable: the rule number cross-references the 20-rule list, the slide/shape name tells you where to click, and the detail describes the fix.

10.6 Auto-fixing findings

Because the audit returns structured data, Claude can loop over it and call edit tools for each finding. Useful follow-up prompts:

Fix every Error-level finding. For small fonts, raise to the minimum 18 pt unless the shape is a footer — in that case leave it. For missing alt text, write a short, concrete description of what the picture shows.
For every Rule 11 finding, move the hard-coded colour to the theme: if the colour is close to an existing accent, swap to the accent; otherwise add a new theme colour and use it.
For Rule 8 title drift, re-align every title to the SlideMaster title placeholder position and report which slides changed.

The audit is also useful as a pre-flight check: run it before exporting to PDF or sending the deck out, and fix the Errors at minimum.

10.7 Limits of the audit

The tool is fast (a 60-slide deck audits in well under a second) but it is rule-based, not visual:

  • Contrast ratio is computed only when the text fill and a solid-colour background can be read from the shape tree. Text over a picture or a gradient is not checked — a human has to look.
  • Chartjunk (Rule 14) is not checked — asking Claude to review the chart visually via get_slide_thumbnail is the current workaround.
  • Story structure (Rule 20) is too judgement-heavy for rules — but the closing-CTA check (Rule 37) catches the most common failure mode: decks that end without an ask. Ask Claude directly for the deeper review: "Does this deck open with context and end with an ask? Where does the narrative break?"
