Chapter 1 — Introduction

Chapter 1Meet VisClaw

VisClaw is a Visio add-in from kalmstrom.com Business Solutions. It opens a chat panel on the right side of your drawing and lets you describe what you want done to the diagram — in plain English — and then does it, directly on the file you already have open.

1.1 What it is

VisClaw is a Microsoft Visio task-pane add-in built with VSTO. It runs inside the Visio process and drives your open drawing through the native Visio object model. Every change it makes is a real Visio edit that you can see happen, undo with Ctrl+Z, and save with the file in the normal way.

Under the hood it talks to Anthropic's Claude language model. You supply your own API key; the add-in does not run through any kalmstrom-hosted service.

1.2 What makes it different

Lives inside Visio

No separate app, no export, no copy and paste. VisClaw drives the drawing Visio already has open.

Deep into the object model

From pages and shapes down to individual ShapeSheet cells. When a high-level tool cannot express what you need, VisClaw reaches into the ShapeSheet directly.

Understands connections

Connectors are first-class. VisClaw can walk the connection graph from any shape, list all connections on a page, and find a shape's neighbours.

Real Visio edits

Every action is a normal Visio edit — undoable with Ctrl+Z, saved with the file in the normal way. Multi-step tool calls can be wrapped in a single undo scope.

Chapter 2 — Installation & Setup

Chapter 2Installation & Setup

Installing VisClaw takes about five minutes: install the add-in, get an Anthropic API key, and paste it into the settings window.

2.1 Install the add-in

  1. Close every open Visio window.
  2. Run the VisClaw installer (VisClaw Setup.exe or the ClickOnce link supplied by kalmstrom.com).
  3. Follow the wizard — accept the prompts to install the Visual Studio Tools for Office (VSTO) runtime if it asks.
  4. Open Visio. On the Home tab you should see a new VisClaw button.
The button is missing? Go to File → Options → Add-ins. At the bottom, change Manage to COM Add-ins, click Go… and make sure VisClaw is ticked. If it is in the Disabled Items list, re-enable it there.

2.2 Get your Anthropic API key

  1. Go to console.anthropic.com and sign in or create an account.
  2. Open the API Keys page.
  3. Click Create key, give it a name such as VisClaw, and copy the key — it starts with sk-ant-.
  4. Add billing information. Anthropic charges pay-as-you-go; there is no monthly subscription required.
Cost in practice. Most VisClaw actions are inexpensive — adding a shape, connecting two boxes, formatting a page uses very few tokens. The exception is "walk the connection graph" on a large diagram, which can return hundreds of nodes.

Chapter 3 — First run

Chapter 3First run

The fastest way to understand what VisClaw feels like is to open an empty drawing and ask it to do three small things in a row.

3.1 Open a blank drawing

In Visio, create a new blank drawing. It doesn't matter which template — a simple blank canvas is fine. Click the VisClaw button on the Home ribbon so the chat panel opens on the right.

3.2 Ask for a simple diagram

Drop three rectangles on this page — one labelled "Input", one "Process" and one "Output" — and connect them in order with straight connectors.

VisClaw calls add_shape three times (with the text set in each call), then add_connector twice to link them. You see each shape and connector appear live. If you don't like the result, Ctrl+Z backs the whole thing out.

3.3 Ask for something that uses the connection graph

List all shapes connected to the Process shape. Then set the fill colour of every one of them to pale blue.

This is where VisClaw earns its keep. It calls get_shape_neighbors to find the connected shapes, then set_shape_fill_color on each one. A purely text-based assistant could not do this without VisClaw's graph-aware tools.

3.4 Ask for a layout change

Lay out the page as a left-to-right flow chart, then fit the page to the shapes.

VisClaw calls layout_page with the flow-chart style and fit_page_to_shapes so the page border closes up around the content.

3.5 Watch out for…

  • Units. Visio pages can be in inches, millimetres, or pixels depending on the template. When VisClaw sets positions or sizes it uses the page's current units — if something lands in the wrong place, check the units on the Design tab.
  • Stencils must be open. To drop a master by name, its stencil needs to be open. Use open_stencil or open the stencil in Visio first.
  • Hidden shapes and layers. VisClaw sees every shape on a page. If you want certain shapes ignored, put them on a layer and ask VisClaw to filter.

Chapter 4 — Driving Visio

Chapter 4Driving Visio

This chapter walks through what VisClaw can actually do, organised by task. Each section gives a short description of the capability, a few example prompts you can paste into the chat panel, and notes on where Visio's own idioms matter.

4.1 Pages

Pages are the top-level container in any Visio drawing. VisClaw treats them as first-class objects you add, rename, resize, and lay out.

  • Add, delete, rename, list. "Add a page called 'Phase 2'" / "Rename the current page to 'Current state'" / "List every page in this drawing."
  • Size and orientation. "Set this page to A3 portrait" / "Make the page US Letter, landscape."
  • Fit to shapes. "Fit the page to the shapes" — tightens the page border around the actual content so there's no empty border.
  • Lay out. "Lay out this page as a top-down tree" / "left-to-right flow" — VisClaw calls layout_page with the matching style.

4.2 Shapes

Every Visio shape is an instance of a master — either a built-in master (rectangle, ellipse, connector) or one from an open stencil (network switch, BPMN activity, room divider). VisClaw works in both modes.

  • Add a shape. "Add a rectangle at (2, 2) with text 'Input'" — the default master is a rectangle; for anything else, see "Masters and stencils" below.
  • Delete, duplicate, copy to another page. "Delete the shape named 'Draft'" / "Duplicate BoxA" / "Copy the process cluster to page 2."
  • Group / ungroup. "Group these three shapes together" / "Ungroup the header group."
  • Position, size, rotate, z-order. "Move BoxA to (3, 1)" / "Make every 'decision' shape 40×40 mm" / "Rotate the arrow 90° clockwise" / "Bring the callout to the front."
  • Align and distribute. "Align these four shapes to the left" / "Distribute the boxes evenly horizontally."

4.3 Formatting

  • Fill colour. "Make every 'bottleneck' shape red."
  • Line colour, weight, pattern. "Set the connector line weight to 2pt" / "Make the border of every decision shape dashed."
  • Arrows. "Put open-arrow heads on both ends of every connector."

4.4 Shape text

Every Visio shape has a text block. VisClaw can set the text and format it independently.

  • Set text. "Set the text on BoxA to 'Customer places order'."
  • Font, size, colour. "Use Calibri 10pt white text on every dark-filled shape."
  • Style and alignment. "Bold the text on every title shape, centre-align it." Styles include bold, italic, underline.

4.5 Masters and stencils

A stencil is a collection of masters you can drop onto a page. Visio ships dozens of built-in stencils (Flowchart, Network, BPMN, Basic, …); you can also author your own.

  • List masters. "What masters are available right now?" — VisClaw reports every master from every open stencil.
  • Open a stencil. "Open the BPMN Basic Shapes stencil." After this, its masters become available.
  • Drop a master by name. "Drop a 'Decision' from the Flowchart Shapes stencil at (4, 3)." Names are matched case-insensitively.
If you don't know the master name. Ask "list the masters in the Network stencil" first. VisClaw returns the full list; you then reference them by exact name.

4.6 ShapeSheet access

Every Visio shape is backed by a ShapeSheet of raw cells — positions, formulas, custom properties, geometry. Most of the time you don't need to touch the ShapeSheet; the typed tools cover what people actually do. But when you need something unusual, VisClaw exposes direct cell access:

  • Read a cell. "What is the value of Geometry1.X1 on BoxA?" — VisClaw calls get_shape_cell.
  • Write a cell. "Set Prop.Status to 'Completed' on every shape in the first row" — set_shape_cell.
Reach for this when the typed tools run out. Examples: setting a user-defined cell, writing a formula that references another shape, forcing a non-standard geometry vertex.

4.7 Shape data

"Shape Data" is the Visio term for the named-property bag on each shape (Prop.Status, Prop.Cost, Prop.Owner, …). VisClaw lists and sets them:

  • List. "What shape data does BoxA have?"
  • Set. "Set Prop.Owner to 'Finance' on every shape in the Finance lane."

4.8 Finding shapes

  • By master. "Find every shape that uses the 'Decision' master."
  • By text. "Find every shape containing the word 'bottleneck'."
  • List on a page. "List all shapes on this page with their positions."
  • Detail. "Tell me everything you can about BoxA" — name, master, position, size, rotation, text, shape data, connected shapes.

4.9 Connectors and connection graphs

Connectors are a Visio specialty. They carry typed endpoint references back to the shapes they connect, so moving either end moves the connector with it. VisClaw treats the connection graph as a first-class object you can query.

  • Add a connector. "Connect BoxA to BoxB with an elbow connector."
  • Enumerate endpoints. "Which shapes does connector 'C1' link?" — VisClaw calls get_connector_endpoints.
  • List connections on a page. "Give me every connection on this page." Returns the list of (connector, begin shape, end shape) triples.
  • Walk the graph. "Walk the connection graph starting from BoxA" — walk_connection_graph does a breadth-first walk.
  • Get neighbours. "What shapes are directly connected to BoxA?" — get_shape_neighbors.
Use case that needs this. "Highlight every shape downstream of the 'Approval' step in red." VisClaw walks the graph from Approval, then recolours everything it reaches.

4.10 Hyperlinks

"Add a hyperlink on BoxA pointing to the design document." Hyperlinks are attached per shape; clicking the shape in a saved drawing navigates to the target.

4.11 Export and save

  • Export a page as PNG. "Export the current page to C:\Exports\flow.png."
  • Export the whole document as PDF. "Save this as PDF at C:\Reports\architecture.pdf."
  • Save the document. "Save."

4.12 Undo scopes

When VisClaw makes a multi-step edit — say, dropping fifteen shapes and wiring them up — it wraps the whole sequence in a single undo scope so Ctrl+Z backs the whole thing out, not the last click. You usually don't have to think about this; VisClaw does it for non-trivial actions automatically.

Chapter 4 — Example recipes

4.13 Example recipes

Build a flow chart from a description

Make a flow chart on a new page called "Order flow" with these steps: Customer places order → Check stock → (branch: in stock → Ship; out of stock → Backorder) → Send confirmation. Use the Flowchart Shapes stencil — terminator for start and end, decision diamond for "Check stock", process rectangles for the others. Connect them with elbow connectors.

On this page, list every shape that has no connections to anything else. Then for every 'Decision' shape with more than two outgoing connectors, tell me which one.

Batch-update shape data

Prepare for print

Set this page to A3 landscape, fit the page to the shapes, then export to PDF at C:\Reports\process-map.pdf.

Chapter 5 — Settings

Chapter 5Settings

VisClaw's settings window is reached from the gear icon in the chat panel header. Fields are deliberately minimal.

5.1 API key

Your Anthropic API key, encrypted with Windows DPAPI and stored under %APPDATA%\VisClaw\. Click Show to reveal it temporarily.

Rotate your key if you share a machine. DPAPI protects keys from other Windows accounts on the same machine, but if you allow another person to use your Windows session, they can use your API key. Generate a new key in the Anthropic console and paste it here.

5.2 Model

Chooses which Claude model answers your messages.

ModelWhen to use
HaikuFast and cheap. Good for routine tasks with small inputs and short replies.
SonnetBalanced default. Handles summarisation, multi-step reasoning and longer drafts.
OpusHighest quality, slower and more expensive. Use for nuanced tone, sensitive material or hard judgement calls.

5.3 Logging

Controls the local log file used for diagnostics.

  • Off — no log written.
  • Errors only (default) — only failures are recorded.
  • Verbose — every tool call, input and result is written to the log.

Log file location: %APPDATA%\VisClaw\visclaw.log. It is a plain text file you can open in Notepad. Document, mailbox or workbook contents are not logged at any verbosity level.

5.4 Panel position

By default the chat panel docks on the right of the main Visio window. You can float it or dock it on the left. The position is remembered per Windows profile.

Chapter 6 — Tips & troubleshooting

Chapter 6Tips & troubleshooting

6.1 Getting the most out of VisClaw

  • Name shapes that matter. VisClaw identifies shapes by name. The default names Visio assigns ("Sheet.17") are not memorable — rename important shapes early so you can refer to them naturally: "connect the Approval shape to the Dispatch shape."
  • Open the stencils you want masters from first. VisClaw cannot drop a master from a stencil that is not open. Say "open the Flowchart Shapes stencil" before asking for flowchart-specific shapes.
  • Let VisClaw walk the graph. For anything that depends on "what is connected to what", prefer a graph-walking prompt ("colour everything reachable from X") over an enumerate-by-hand prompt. The graph tools are faster and far less error-prone.
  • Use undo liberally. Every VisClaw action is a normal Visio edit. Ctrl+Z works. Multi-step actions are wrapped in single undo scopes so one undo backs the whole operation out.

6.2 Common pitfalls

SymptomCauseFix
"Master not found"The stencil with that master is not open.Open the stencil first ("open the BPMN stencil"), or ask VisClaw to list available masters.
Shape positions look wrongPage units differ from what you assumed.Check page units on the Design tab. Ask VisClaw to set the size in the unit you want.
Connector stuck on the wrong endConnector was attached by coordinates, not by shape name.Ask VisClaw to reconnect it by shape name — endpoint connections stay attached when shapes move.
ShapeSheet write rejectedCell name wrong or cell is read-only in that shape type.Ask VisClaw to read the cell first to see if it exists.

6.3 Cost and token tips

  • Adding shapes and connectors is cheap. Walking a 500-shape connection graph is not — scope the walk with a depth limit when the diagram is large.
  • Haiku is usually sufficient for routine editing. Reach for Sonnet when you want the AI to reason about the diagram as a whole (auditing, restructuring, describing).

6.4 Bug-report helpers

Two built-in tools make it easy to gather the build fingerprint and capability surface when you need to report an issue:

  • get_version_info — ask "what version of VisClaw am I running?" and VisClaw replies with add-in name, auto-refreshed build version, the Visio host version, OS, and .NET runtime. The same version number is always visible in the top-right corner of the Settings dialog, so you can grab it without sending a chat turn.
  • list_tools — ask "which VisClaw tools mention 'layer'?" and it filters the full tool catalogue by any substring of the name or description. Useful when you're not sure whether a capability exists, or when you want to compare what's available against the online docs.

Chapter 7 — Advanced operations

Chapter 7Advanced operations

Chapter 4 covers the day-to-day surface: pages, shapes, connectors, data, stencils. This chapter covers the operations people reach for less often — masters authoring, themes & quick styles, data-graphics refresh, containers and callouts, hyperlinks, and layers. Everything here still works through the chat prompt; VisClaw picks the right tool.

7.1 Themes, quick styles, theme variants

Visio themes colour a whole drawing consistently. VisClaw lists, applies, and switches them from chat.

  • List available themes. "What themes does this drawing support?" Returns the built-in + custom themes.
  • Apply a theme. "Apply the 'Organic' theme to the current page." Every themed shape re-colours; shapes set to "use theme colour" update automatically.
  • Apply a variant. "Switch to the third variant of 'Organic'." Variants are the colour/accent subsets each theme ships.
  • Apply a quick style. "Set shape 'Server 1' to quick style 'Subtle effect 2'." Quick styles are the named fill/line/shadow combinations from the Design tab.

Graceful degradation. Some older Visio builds don't expose the Theme API via COM dispatch. VisClaw detects that at call time and returns a "this Visio build doesn't support X" message rather than crashing the pane.

7.2 Masters and stencil authoring

  • Create a master from the current selection. "Turn this selection into a new master called 'Standard server'." The master is added to the document stencil and the selection becomes an instance of it.
  • List masters. "What masters are in this document's stencil?" / "List every master in the 'Basic Flowchart' stencil."
  • Drop a master. "Drop a 'Decision' shape at 4,6 inches."
  • Drop a legend. "Drop a legend for the data graphic at the top-right corner."

7.3 Containers, callouts, and lists

Containers are Visio's grouping primitive that behave as a single logical unit — move the container, the children follow.

  • Drop a container. "Drop a container around the three selected servers and label it 'Production tier'."
  • Add a shape to an existing container. "Add 'Server 4' to the 'Production tier' container."
  • Remove from container. "Remove 'Server 2' from 'Production tier' but keep it on the page."
  • Drop a callout. "Add a callout pointing to 'Router' that says 'Failover pair — active/standby'."

7.4 Connector glue

Glue is Visio's word for the endpoint connection between a connector and a shape. VisClaw manages glue directly.

  • Glue an endpoint. "Glue the end of connector C7 to the right side of 'Router'."
  • Unglue an endpoint. "Unglue both ends of connector C7." Uses a FormulaU rewrite under the hood (not Cell.GlueTo(null), which throws on some Visio builds).
  • Trace glue. "What's glued to 'Router's right side?" Lists every connector touching the named shape, with direction.

7.5 Hyperlinks and shape data

  • Add a hyperlink. "Add a hyperlink on 'Design doc' shape pointing to https://wiki/design with sub-address 'architecture'."
  • List hyperlinks. "What hyperlinks are on the 'Infrastructure' page?"
  • Shape data rows. "List shape data on 'Server 1'" / "Set its 'Owner' property to 'Infra team'" / "Bulk-set 'Environment' to 'prod' across every server shape."

7.6 Data graphics and refresh

  • Refresh all data recordsets. "Refresh the data bindings." VisClaw walks every DataRecordset and calls Refresh, silently skipping any that don't expose the method on older Visio builds.
  • Apply a data graphic. "Apply the 'Availability' data graphic to every server shape on this page." (Roadmap — not yet implemented; pending confirmation of the COM surface across Visio builds.)

7.7 Layers

Layers organise visibility, print, and lock state independently of shape geometry.

  • List layers. "What layers does this page have?"
  • Create / delete. "Add a new layer called 'Annotations'." / "Delete the 'Scratch' layer."
  • Assign shapes. "Put every callout on the 'Annotations' layer."
  • Visibility and lock. "Hide the 'Annotations' layer" / "Lock the 'Network' layer so nobody edits it by accident."
  • Set active layer. "Make 'Annotations' the active layer so new shapes land there."

7.8 Selection and navigation

  • Select by pattern. "Select every shape whose name contains 'Server'." / "Select all shapes on the 'Annotations' layer."
  • Scroll to a shape. "Scroll to and centre 'Router'."
  • Zoom. "Zoom to fit the whole page" / "Zoom to 150%." Uses visFitPage for fit-to-page (not the older visFitToShapes, which newer Visio builds rejected).

7.9 Drawing primitives

Chapter 4 covers dropping masters. VisClaw also drops raw geometric primitives for sketches.

  • Rect, ellipse, line. "Draw a rectangle from 0,0 to 4,3 inches" / "ellipse centred on 5,5 inches, 2 inches wide, 1 inch tall" / "line from 1,1 to 6,1."
  • Bezier curve. "Draw a bezier curve through these four points: (1,1), (3,5), (5,1), (7,3)."
