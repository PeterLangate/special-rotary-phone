Chapter 1 — Introduction

Chapter 1What the Claw Suite is

The Claw Suite is six Microsoft Office add-ins from kalmstrom.com Business Solutions. They share a codebase and a philosophy: let you drive your file in plain English, backed by a modern language model, without leaving the app you already have open.

1.1 The products

ProductHostWhat it drives
PointClawPowerPointSlides, shapes, animations, media, themes, speaker notes, deck audits.
ExClawExcelRanges, formulas, tables, pivot tables, charts, formatting, imports and exports.
WorClawWordParagraphs, styles, headings, tables, TOC, revisions, mail merge, content controls.
OutClawOutlookMail, calendar, contacts, rules, search, archive — driven through Outlook Redemption.
VisClawVisioPages, shapes, connectors, stencils, ShapeSheet cells, connection graphs, layout automation.
ProjClawProjectTasks, resources, assignments, dependencies, baselines, earned value, critical path, calendars.

Each one ships independently. You install only the Claws you need, and each adds a button to its host's ribbon that opens a chat panel.

1.2 What they have in common

Real edits, not suggestions

Every Claw drives the live object model of its host. Changes appear in your document as they happen and can be undone with Ctrl+Z.

Plain language, specialist tools

You do not learn command syntax. The AI picks the right purpose-built tool — the suite ships more than six hundred of them.

Bring your own API key

Anthropic Claude for conversation; Google Gemini for image and video generation (PointClaw only). Keys are stored locally with Windows DPAPI.

One shared code spine

A single chat pane, a single settings window, a single tool-dispatch plumbing layer. A fix in the shared code lands in all six add-ins on the next build.

1.4 Who the suite is for

  • Knowledge workers who live inside Office and want an assistant that actually works on their files, not just beside them.
  • Consultants and analysts producing decks, reports, spreadsheets and client emails under time pressure.
  • Teachers, trainers, and presenters who iterate on the same material often.
  • Power users who want reliable COM-level operations without writing VBA.
Not a Microsoft product. The Claw Suite is not built on or licensed from Microsoft. It uses only publicly documented Office object models and Outlook Redemption for Outlook-specific depth. You pay Anthropic and (optionally) Google directly for the API calls.

Chapter 2 — Shared architecture

Chapter 2Shared architecture

All six Claws are VSTO desktop add-ins written in C#, targeting .NET Framework 4.7.2 and AnyCPU. They are not independent products sharing a marketing name — they are one codebase with six host-specific front ends.

2.1 The seven-piece skeleton

Each Claw follows the same seven-piece layout. If you can read one, you can read any of them:

PieceLives inResponsibility
ThisAddIn.csEach ClawVSTO entry point; registers the pane per host-window.
Ribbon.cs + Ribbon.xmlEach ClawRibbon button(s) that toggle the chat pane.
Api/{Host}Context.csEach ClawTyped accessors for the active Presentation / Document / Workbook / Inspector.
Api/{Host}AgentToolBase.csEach ClawBase class for tools; UI-thread marshalling, error surfacing.
Api/{Host}ToolDispatcher.csEach ClawRegisters every tool, routes tool calls, injects context.
Api/{Host}SystemPromptBuilder.csEach ClawHost-specific system prompt describing the environment.
UI/ChatHostBindings.csEach ClawPer-host glue: history storage, context block builder, settings window.

2.2 The shared libraries under Admin/

Everything that is not host-specific lives in X:\Source\Admin\ and is linked (not copied) into each Claw's csproj. A fix in shared code lands in every add-in on the next build.

LibraryWhat it contributes
kSharedCSharpCore error-handling primitives (kS.kErrorMode, kS.kGlobalErrorHandler), null-safety helpers, DPAPI key store.
kSharedCSharpUIThe chat pane (ChatPane.xaml), the settings window, WPF theming, the markdown renderer, the choice prompter. Host-agnostic — parameterised through IChatHostBindings.
kSharedCSharpAgentThe LLM plumbing: ClaudeClient, GeminiChatClient, OpenAIChatClient, tool-dispatch interfaces, self-test engine, JSON helpers, COM reflection helpers, the shared TemporaryComDocumentScope and BuiltInPropertyHelper.
RedemptionOutlook Redemption binaries and loader (used by OutClaw only).
The reuse-first rule. When a fix is needed in shared code, it is fixed in shared code. Project-local workarounds are never an acceptable answer. Every fix benefits all six Claws.

2.3 The chat pane is one WPF control

Every Claw renders the same ChatPane.xaml control from kSharedCSharpUI. The per-host glue is thin — a couple of dozen lines implementing IChatHostBindings — but the chat loop, streaming renderer, usage stats, settings access, markdown rendering and choice prompting are all in the shared library.

What the chat pane cares about from its host:

  • How to store chat history (CustomXMLParts for PointClaw / ExClaw / WorClaw; JSON on disk for OutClaw, because Outlook items do not have custom XML parts).
  • How to build a context block (the current state of the document, stapled to the system prompt on every turn).
  • Which tool dispatcher to use.
  • Which system prompt to issue.

2.4 The tool pattern

Every tool is a C# class inheriting from the Claw's host-specific *AgentToolBase. It declares four things:

  1. Name — the tool identifier the LLM uses (e.g. add_slide, create_table).
  2. Description — a one-to-three sentence description the LLM reads to decide when to call.
  3. InputSchema — a JSON Schema fragment describing the parameters.
  4. Execute — the work. Typically a Reader or Runner inner class that the base class marshals to the UI thread.

Optionally the tool implements IToolSelfTest.BuildTestCases() to supply headless self-test cases. Those run through the shared ToolSelfTestEngine with a standard driver script per host (see section 2.6).

2.5 The backend is pluggable

Conversation runs through IAgentClient. Three implementations ship today:

BackendEndpointTool calls
ClaudeClientAnthropic Messages APItool_use + prompt caching
GeminiChatClientstreamGenerateContent?alt=ssefunctionCall
OpenAIChatClientChat Completionstool_calls

Image and video generation (PointClaw only) go through a separate GeminiClient that wraps Imagen and Veo. AgentClientFactory.Create picks the right backend from the persisted user setting; users switch between them from the Chat Backend dropdown in Settings.

2.7 The deploy story

  • Development: F5 in Visual Studio. VSTO launches the host with the add-in registered under HKCU\Software\Microsoft\Office\<App>\Addins\.
  • Release: ClickOnce publish from Visual Studio, signed with _TemporaryKey.pfx (until a real code-signing cert is wired up).
  • All six Claws share PointClaw's packages/ folder — NuGet versions are bumped in one place and the others inherit.

Chapter 3 — The products

Chapter 3The products

Each Claw has its own detailed user manual. This chapter is the one-page summary of each — what they feel like to use, what they are especially good at, and where their individual manuals go next.

3.1 PointClaw — for PowerPoint

PointClaw is the original, and the one with the deepest "generative" surface. It generates images with Google Imagen and short video clips with Google Veo and drops them straight onto the slide. It drives the full animation stack: triggered animations, motion paths, chart-series animation, slide transitions. It runs a built-in deck audit that scores slides against a house design opinion and fixes what it finds.

  • Create and open presentations; start from a template; save and export PDF / video.
  • Every shape type: text boxes, tables, SmartArt, charts, pictures, word art, action buttons, countdown timers, YouTube embeds.
  • Full theme control: colours, fonts, effects, slide masters, handout and notes masters.
  • Accessibility checks; screen capture; live slide thumbnails for visual feedback.

See the PointClaw User Manual for details.

3.2 ExClaw — for Excel

ExClaw treats structured objects — tables, pivot tables, named ranges — as first-class citizens. They are easier for the AI to reason about than bare A1:F500 addresses, and ExClaw leans on them heavily.

  • Read and write values, formulas, and array formulas; fill series; import / export CSV.
  • Create and manage tables, named ranges, pivot tables, and pivot calculated fields.
  • Insert and modify charts, shapes, pictures, hyperlinks.
  • Conditional formats, data validation, sort and filter, freeze panes, protection.
  • Run existing macros; switch calculation mode; refresh data connections.
  • Template inspection: silently open an .xltx to report its sheet inventory, named ranges and tables before you commit to using it.

See the ExClaw User Manual for details.

3.3 WorClaw — for Word

WorClaw is the most structurally aware Claw. It works with paragraphs, headings, styles, sections, bookmarks, cross-references, fields, footnotes, and content controls — the scaffolding of real documents, not just strings of text.

  • Read paragraphs, headings, selection; find by text, style, or field.
  • Full style lifecycle: create, modify, list, apply, find-by-style.
  • TOC, table of figures, cross-references, captions, indexes, footnotes, endnotes.
  • Track changes, revisions, comments, compare documents.
  • Mail merge (simple and multi-file), content controls, building blocks, templates.
  • Headers, footers, page setup, section starts, columns, page numbering.
  • Font analysis across an entire document; template file inspection.

See the WorClaw User Manual for details.

3.4 OutClaw — for Outlook

OutClaw is the odd one out. Outlook's security model aggressively blocks normal COM access to mail items — the "Outlook Object Model Guard" fires a user prompt every time an add-in tries to read the body of a message or send mail. OutClaw sidesteps this entirely by using Outlook Redemption: RDO / Safe*Item objects that bypass the guard, run on secondary threads, and expose raw MAPI properties.

  • Mail: read, send, reply, forward, archive, categorise, flag, move, delete.
  • Calendar: create, update, list meetings and appointments; free/busy lookup.
  • Contacts: create, update, list, search.
  • Rules: create, modify, enable/disable; search and summarise.
  • Attachments: inspect, save, add, remove.
  • Folders: enumerate, move items, count messages.
  • Search across all mailboxes and stores.
A per-user session is required. Outlook enforces a singleton model per Windows profile — a foreign-session Outlook will block OutClaw's driver. See the OutClaw manual for specifics.

3.5 VisClaw — for Visio

VisClaw is the diagram-and-shape Claw. Visio's object model is deep but idiosyncratic: shapes are backed by stencil masters, every shape has a ShapeSheet of raw cells, connectors carry endpoint references back to the shapes they connect, and pages form a small network of their own. VisClaw exposes that surface rather than hiding it, then gives the AI enough layout primitives to do the boring work.

  • Pages. Add, delete, rename, list, set size, set orientation, fit-to-shapes.
  • Shapes. Add, delete, duplicate, copy to another page, group, ungroup, rotate, z-order, align, distribute.
  • Formatting. Fill colour, line colour, line weight, line arrows, line pattern.
  • Text on shapes. Set text, font, size, colour, style, alignment.
  • ShapeSheet access. Get / set individual ShapeSheet cells for the cases a high-level API cannot express.
  • Shape data properties. List and set the name/value pairs Visio calls "shape data".
  • Masters and stencils. List masters, open a stencil, drop a named master onto a page.
  • Finding. Search by master, by text; fetch full shape detail; list all shapes on a page.
  • Connectors and connection graphs. Add a connector, enumerate endpoints, list all connections on a page, walk the graph from any shape, fetch a shape's neighbours.
  • Export and save. Page PNG, whole-document PDF, save.
  • Undo scopes. Begin / end an undo scope so the AI can treat a multi-step edit as a single undo step.

See the VisClaw User Manual for details.

3.6 ProjClaw — for Project

ProjClaw is the only Claw whose primary data is a schedule rather than a document. Tasks, resources, assignments, dependencies, calendars, baselines, and earned value are all first-class objects. ProjClaw does not try to replace Project's own scheduling engine — it drives it through the native COM object model.

  • Tasks — full lifecycle. Add (single + bulk), delete, list, find by name, get detail, set any field, indent / outdent, move through the hierarchy tree.
  • Dependencies. Add, remove, chain-link, fetch predecessors / successors, get slack, identify the critical path.
  • Task properties. Milestone, task type, notes, priority, constraint, deadline.
  • Progress tracking. Set percent complete, actual start, actual finish, actual work.
  • Resources. Add, delete, list, set fields, set max units, set cost rate, set type, level resources, find over-allocated ones.
  • Assignments. Assign, bulk assign, remove; set units or work; list per task.
  • Calendars. List, set working / non-working time, set the project calendar.
  • Baselines and variance. Save a baseline, compare to baseline, get variance, get earned value.
  • Costs. Per-task cost, fixed cost, project cost.
  • Status reporting. Set the status date, generate a status report.
  • Views, filters, custom fields. Switch views, apply filters, read / write custom field values.
  • Export. PDF and Excel.
  • Undo scopes. Same pattern as VisClaw — wrap multi-step edits in a single undo step.

See the ProjClaw User Manual for details.

3.7 How they work together

Each Claw is a separate add-in, so they install and run independently. But they are built to play nicely:

  • Consistent chat experience. The chat panel looks and feels identical across all six hosts. Once you know how to drive one Claw, you know how to drive the others.
  • Shared settings window. Same API key fields, same model picker, same backend dropdown. Same partials in the user manuals.
  • One API key, many uses. An Anthropic key works across all six Claws. You do not pay per-add-in.
  • Cross-host workflows are manual for now. ExClaw does not talk to WorClaw today. If you want to flow data between hosts, Windows clipboard or saved intermediate files are the current bridge.

3.8 Ribbon footprint

Each Claw adds a minimal ribbon footprint:

  • PointClaw / ExClaw / WorClaw / VisClaw / ProjClaw: one toggle button on the Home ribbon that opens or closes the chat pane.
  • OutClaw: eight buttons — mail, calendar, contacts, rules, search, archive, attachments, settings — each opens the chat pane pre-scoped to that task.

Chapter 4 — How it compares

Chapter 4How the Claw Suite compares

The Claw Suite is not the only way to get an AI into Office. Three categories of alternative exist today — Microsoft's own Copilot and partner add-ins, Anthropic's Claude add-ins, and the open-source Model Context Protocol (MCP) servers that expose Office operations to external AI clients. Each trades something off; knowing where they differ helps you pick the right tool for each job.

4.1 Versus Anthropic's Claude add-ins

Anthropic publishes its own Claude for PowerPoint / Excel / Word add-ins through Microsoft AppSource. They also put Claude inside Office, but the underlying technology is very different.

Claw SuiteClaude for Office (Anthropic)

Reach into OfficeFull native object model: triggered animations, motion paths, SmartArt, pivot tables, content controls, mail merge, Redemption-level Outlook access, deck audits, PDF / video exportLimited to what Office.js exposes — mainly generating, rewriting and summarising content

Rule of thumb. Claude for Office is broader and runs everywhere, but stays on the surface. The Claw Suite trades cross-platform reach for depth — it can drive internals that a web-sandboxed add-in cannot touch.

4.3 Versus Microsoft Copilot and Agent 365

Microsoft's own strategy is different from anything described above. It has two distinct layers worth understanding.

Microsoft 365 Copilot

Copilot is Microsoft's consumer-facing agent. It lives inside every Microsoft 365 app, works against your tenant's data, and is subject to your tenant's compliance boundaries. For most enterprise users this is the obvious starting point. Its reach is broad but shallow: like Claude for Office, it operates through the Office.js-era APIs and cannot touch features that require COM-level control.

Microsoft Agent 365

Agent 365, announced in November 2025 and generally available from May 2026, is a very different thing. It is not an agent — it is the control plane for agents running in a Microsoft 365 tenant. Its job is to govern, identify, and observe the agents that Copilot, partners, and in-house developers bring into the organisation.

They are complementary, not competitive. Agent 365 is designed to bring agents under management — it will identify the Claws as "shadow agents" in a tenant that has it enabled. From Microsoft's point of view that is a feature, not a bug: you deploy Agent 365 precisely because you want to see (and optionally limit) what agents are doing.

What this means for the Claw Suite

  1. For individual and small-team users today, the model does not change. Agent 365 is an M365 E7 SKU aimed at large enterprises. A consultant, teacher or analyst using the Claws is not its audience.
  2. For future enterprise adoption, integration will matter. A Claw add-in that registers itself as an Entra agent, emits telemetry via the Observability SDK, and respects tenant policy would be a governable agent in an Agent-365-enabled tenant. Without that, it is a shadow agent that IT may block. See Chapter 5 for the roadmap.
  3. Capability still matters. Microsoft's bet is that governance will dominate the enterprise buying decision. The Claw Suite's bet is that capability — a tool surface north of six hundred — dominates the individual-user decision. Both can be correct simultaneously at different market tiers.

Chapter 5 — Roadmap

Chapter 5Roadmap

The Claw Suite is a living codebase. New tools land often; occasionally an architectural piece gets promoted into shared infrastructure. This chapter sketches the current direction — what is on the near-term radar, and what the Agent 365 landscape implies for longer-term positioning.

5.1 Near-term priorities

  • Runtime verification. Many new tools are shipped in waves, compile-verified but not yet self-tested against live Office. Running the per-host Run-SelfTest*.ps1 harnesses regularly catches COM signature surprises the C# compiler cannot.
  • Consistency sweep. The style rules (use ToolArgs.RequireX for required inputs, ToolTestCase.ErrorPathCall for empty-input guards, action-verb descriptions) are applied to every new tool, and older tools are being retrofitted as they are touched.
  • Per-Claw manuals. All six Claws ship full user manuals built from the shared chapter template; duplicated boilerplate (install steps, key-entry steps, button-missing note, chat pane anatomy, tool-call loop diagram) lives in Documentation\Human\_shared\partials\ and is rendered into each manual at build time.

Chapter 5 — Roadmap

5.3 Long-term: enterprise governance integration

The emergence of Microsoft Agent 365 (see Chapter 4) changes the enterprise conversation. An agent that wants to run inside a regulated, Agent-365-managed tenant needs three things:

  1. An Entra agent ID — registration with Microsoft Entra so IT can see the agent exists, scope its access, and apply conditional access policies.
  2. Observability SDK integration — emit telemetry so the tenant's Agent 365 dashboards can track what each agent is doing.
  3. Policy-compatible identity and key management — move away from DPAPI-stored personal keys toward Entra-managed secrets or a centrally-approved key rotation scheme.

None of that is free. It changes the deployment shape from "install a VSTO add-in, paste your API key" to "onboard an agent into your tenant, wait for IT sign-off, accept the policies." The Claw Suite's current audience — individuals, small teams, and consultants — does not pay for that complexity; it is still better for them to pay Anthropic directly.

But if a large enterprise ever wants to deploy the Claws under Agent 365 governance, the work above is the path. It is a deliberate deferral, not an accidental gap.

Two audiences, two products? It is possible the right answer is to keep today's Claws as the consumer / SMB offering and fork an "Enterprise Edition" that accepts the Agent 365 constraints in exchange for being deployable at scale. Both bets can be right simultaneously: capability dominates the individual-user decision; governance dominates the enterprise-IT decision.

5.4 How to influence the roadmap

  • Send a feature request. Each per-Claw manual lists the support contact. The roadmap is driven by real user requests, not speculative planning.
  • Log evidence. When a tool misbehaves, set logging to Verbose (Settings → Logging) and send the %APPDATA%\<Claw>\<claw>.log excerpt with your API key redacted. Runtime evidence is worth more than a symptom description.
  • Tell kalmstrom.com which hosts matter to you. If your workflow is Excel-first, the roadmap should reflect that.
