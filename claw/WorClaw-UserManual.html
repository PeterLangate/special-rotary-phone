Chapter 1 — Introduction

Chapter 1Meet WorClaw

WorClaw is a Word add-in from kalmstrom.com Business Solutions. It opens a chat panel on the right side of your document, and from that panel you can ask an AI to draft, edit, structure and review your document — using plain English instead of menus, ribbons and keyboard shortcuts.

1.1 What it is

WorClaw is a Microsoft Word add-in built with VSTO (Visual Studio Tools for Office). It runs inside the Word process — not as a separate app — and drives your open document through the native Word object model. Every change it makes is a real Word edit that you can see happening in front of you, undo with Ctrl+Z, and save with the file in the normal way.

Under the hood it talks to Anthropic's Claude language model for conversation and reasoning. You supply your own Anthropic API key — the add-in does not run through any kalmstrom-hosted service.

1.2 What makes it different

Lives inside Word

No separate app, no browser, no copy and paste. The add-in talks directly to the document you already have open.

Works on real documents

Every action is a normal Word edit. You can undo, redo, tweak or throw away any result just like your own work.

One hundred specialist tools

More than one hundred purpose-built tools for paragraphs, headings, tables, styles, revisions, mail merge, content controls, building blocks, fields, TOC, cross-references, captions and export — no HTML scraping, no screenshot tricks.

Destructive ops confirm

Deleting a selection triggers an explicit yes/no prompt. WorClaw will not silently erase content.

Chapter 2 — Installation & Setup

Chapter 2Installation & Setup

Installing WorClaw takes about five minutes: install the add-in, get an Anthropic API key, and paste it into the settings window.

2.1 Install the add-in

  1. Close every open Word window.
  2. Run the WorClaw installer (WorClaw Setup.exe or the ClickOnce link supplied by kalmstrom.com).
  3. Follow the wizard — accept the prompts to install the Visual Studio Tools for Office (VSTO) runtime if it asks.
  4. Open Word. On the Home tab you should see a new WorClaw button.
The button is missing? Go to File → Options → Add-ins. At the bottom, change Manage to COM Add-ins, click Go… and make sure WorClaw is ticked. If it is in the Disabled Items list, re-enable it there.

2.2 Get your Anthropic API key

Claude is the language model that powers the conversation. You need your own key so that usage is billed to you, not shared.

  1. Go to console.anthropic.com and sign in or create an account.
  2. Open the API Keys page.
  3. Click Create key, give it a name such as WorClaw, and copy the key — it starts with sk-ant-.
  4. Add billing information. Anthropic charges pay-as-you-go; there is no monthly subscription required.
Cost in practice. Short edits and find-and-replace are cheap. Drafting long sections or restructuring a 40-page document can use more tokens. Use Haiku for routine work, Sonnet for drafting, Opus for nuanced review.

3.3 How WorClaw thinks

Every instruction goes to Claude together with a short description of your document context (file name, page count, selection). Claude breaks the request into one or more tool calls — precise operations like insert a paragraph after paragraph 3 with style "Heading 2" — which WorClaw then runs against the live document. After each batch, Claude decides whether it is done or needs to do more.

Things to know:

  • WorClaw can see paragraph indices, heading structure and selection location. It does not always have the full text of every paragraph in context — it asks for what it needs, when it needs it.
  • Insert operations place content at specific positions: start, end, selection or after paragraph N. If you do not say where, WorClaw defaults to the end of the document.
  • Deletions confirm. WorClaw will not delete a selection without asking.

3.4 Undo is your friend

Every edit WorClaw makes is a normal Word edit, so Ctrl+Z undoes it. If an instruction goes wrong, undo your way back and rephrase. WorClaw is not destructive unless you asked for it.

Exception: export. Exporting to PDF writes a new file to disk. Undo in Word does not delete the PDF — you would need to delete it from Explorer yourself.

Before WorClaw can edit or restructure, it needs to see what is already there. Four read-only tools cover that: document metadata, the current selection, the heading outline and the paragraph list.

Returns metadata about the active document. No arguments.

How long is this document — pages and words? Who is listed as the author?

4.5 File-level tools

WorClaw can also drive Word at the file level — the same operations you would reach through the File menu — from the chat pane:

  • Create a new document. "Create a new blank document." Opens a fresh Word window next to the current one. Add a save path for immediate save to disk.
  • Start from a template. "Create a new document from C:\Templates\Report.dotx." Word opens the template as an untitled copy; the template file itself is unchanged.
  • Open an existing document. "Open C:\Docs\LastQuarter.docx." The file opens in a new window; your chat pane continues driving the document it started in.
  • Analyze fonts. "What fonts does this document use?" — analyze_fonts tallies every font across every paragraph and warns when a document mixes more than two families. Read-only.
  • Inspect a template before you use it. "What's in C:\Templates\Report.dotx?" — get_template_file_info opens the template silently, reports its page count, sections, custom styles, bookmarks and content controls, then closes it. get_template_info gives a lighter summary when you just need the style count.
Chat pane stays put. When WorClaw creates or opens another document, the new window opens alongside yours but the chat pane remains bound to the document it started in. Open a second chat pane in the new window if you want the AI driving both.

Chapter 5 — Find, replace and delete

Chapter 5Find, replace and delete

Three tools cover search-and-edit: find_text, replace_text and delete_selection. The first is read-only. The other two modify the document — and delete_selection always confirms before acting.

5.1 find_text

Finds all occurrences of a query string. Returns the total count and the first few match locations (paragraph index and offset within the paragraph).

How many times does "forecast" appear? Show me where the first five occurrences are.

Use find_text before replace_text when you are not sure about the scope. A read is always cheaper and safer than a write.

5.2 replace_text

Find-and-replace across the whole active document. Returns the number of replacements.

Replace every occurrence of "Q2" with "second quarter".
Case and whole-word. By default WorClaw respects Word's find-and-replace defaults — case-insensitive, match anywhere. Ask for case sensitivity or whole-word matching in your instruction ("match case", "whole words only") and WorClaw will pass those flags through.
No preview. replace_text does not show you a diff before applying. Always run find_text first for anything risky, or be ready to Ctrl+Z.

5.3 delete_selection

Deletes whatever the active Word Selection currently covers. Destructive — WorClaw always confirms before calling this tool.

The confirmation flow:

  1. You (or the selection in Word) identify the text to delete.
  2. You ask WorClaw to delete the selection.
  3. WorClaw summarises what it is about to do and asks Delete this selection? Yes / No.
  4. You answer. "Yes" deletes. Anything else cancels.

Undo works: Ctrl+Z restores deleted content. But it is still a destructive edit — the confirmation is there for a reason.

5.4 Example recipes

Terminology sweep

Find every occurrence of "customer", "client" and "user" — tell me the counts. I want to pick one term and replace the others.

Cleanup

Replace double spaces with single spaces. Then replace straight quotes with curly quotes throughout the document.

Targeted delete

I've selected a paragraph I want removed. Delete the selection.

Chapter 6 — Inserting and structuring

Chapter 6Inserting and structuring

The everyday editing tools put new content and structure into your document: insert_text, insert_paragraph, insert_bookmark and apply_style cover most of what you would do from the Home ribbon, and four citation tools — add_bibliography_source, list_bibliography_sources, insert_citation and insert_bibliography — handle the References ribbon.

6.1 insert_text

Inserts text at a specified position.

PositionMeaning
startVery beginning of the document.
endVery end of the document (default if unsure).
selectionCurrent Word Selection — inserts at the cursor or replaces the selected range.
after_paragraphRight after the paragraph at the given 1-based index.
Insert "Draft — do not circulate" at the very start of the document.
After paragraph 3, insert a single sentence that introduces the bulleted list that follows.

6.2 insert_paragraph

Inserts a new paragraph. after_paragraph is either a 1-based index or the string "end".

Insert a new paragraph after paragraph 7 that summarises the previous section in one sentence.
Text vs. paragraph. insert_text inserts characters — it does not always create a new paragraph. insert_paragraph always starts a new paragraph with its own paragraph mark. When the distinction matters (for example, when a style is applied per-paragraph), reach for insert_paragraph.

6.3 insert_bookmark

Inserts a bookmark at the current selection. Useful for cross-referencing later. The current version supports position="selection" only.

At my current cursor position, insert a bookmark called "approvals".

6.4 apply_style

Applies a named Word style to the paragraph at a given 1-based index.

Apply "Heading 2" to paragraph 12.
Make paragraphs 4, 8 and 15 into Heading 2s.

WorClaw uses the exact style names from the document's style set. The most common ones:

  • Normal — default body text.
  • Heading 1, Heading 2, Heading 3, Heading 4 — outline levels.
  • Title, Subtitle — document-level headings.
  • Quote, Intense Quote — pull-out quotations.
  • List Bullet, List Number — list items.
Custom styles. If your document template adds custom styles (for example Report Heading 1), apply_style can use those too — pass the exact name as it appears in Word's Styles pane.

6.5 Citations and bibliography

Four tools let WorClaw drive Word's built-in citation engine — the same machinery behind the References ribbon. The rendered style (APA, MLA, Chicago, IEEE and so on) is controlled by Word's References → Style dropdown; WorClaw does not override it, so the same data renders differently as soon as you pick another style.

ToolWhat it does
add_bibliography_sourceRegisters a new source (book, article, website…) in the document's source list.
list_bibliography_sourcesShows every source already stored in the document, with its tag, type, title and author.
update_bibliography_sourcePatches specific fields (title, year, publisher…) on an existing source without having to re-enter the others.
delete_bibliography_sourceRemoves a source by tag. Any remaining citations for that tag render as (ErrorNoSource).
insert_citationDrops an inline citation — e.g. (Smith, 2024, p. 42) — at the cursor, referencing an existing tag.
insert_bibliographyDrops the full, rendered bibliography at the cursor (typically on the last page).
set_bibliography_styleSwitches the document's citation style (APA / MLA / Chicago…); every existing citation re-renders on the next update.

Typical flow

The four tools are designed to be used in order:

  1. Register the source with add_bibliography_source — give it a short, unique tag like Smith2024. You can repeat this for every source in your document.
  2. Cite inline with insert_citation, passing the same tag. Repeat as many times as the source is referenced.
  3. Render the bibliography with insert_bibliography at the end of the document — Word will list every source you registered.
Register a book by Jane Smith called "Modern Word Automation", published by Acme Press in 2024, with tag Smith2024.
Insert a citation for Smith2024 at my cursor, pointing to pages 42–45.
Go to the end of the document and insert the bibliography.
Tags are the contract. Every citation references its source by tag. If you make a typo — Smthi2024 instead of Smith2024 — Word will still insert the citation field, but it renders as (ErrorNoSource). Call list_bibliography_sources if you are not sure which tags exist.

Source types and fields

add_bibliography_source requires tag, source_type and title. The other fields are optional and vary by type:

Chapter 7 — Tables

Chapter 7Tables

Two tools cover tables: one inserts a new table, one sets a single cell. Together they let WorClaw build tables paragraph-by-paragraph or cell-by-cell.

7.1 insert_table

Inserts a table in the active document. By default the table is placed at the end of the document. An optional header_row flag marks the first row as a header.

ParameterNotes
RowsRequired. Total number of rows including any header.
ColumnsRequired. Number of columns.
LocationOptional. Defaults to end of document.
header_rowOptional. When true, the first row is given the table header style.
At the end of the document, insert a 4-column by 6-row table with a header row. Columns: Metric, Q1, Q2, YoY change.

7.2 set_table_cell

Sets the text of a single cell in a table. All indices are 1-based.

ParameterMeaning
table_index1-based position of the table in the document (the first table is table 1).
row1-based row index.
column1-based column index.
textThe new cell text. Replaces whatever is currently in the cell.
In the first table, set row 1 to: Metric, Q1, Q2, YoY change. Then fill the data rows with the values I gave you earlier.
No auto-expanding tables. If you ask WorClaw to fill more rows than the table has, it will set as many as it can and tell you. To grow the table, use insert_table again with the larger size, or add rows manually in Word first.

Chapter 8 — Review & export

Chapter 8Review & export

Two tools cover the end-of-document workflow: toggle_track_changes for collaborative review and export_pdf for producing a shareable copy.

8.1 toggle_track_changes

Turns Word's Track Changes on or off in the active document.

Turn on track changes so I can see what my next edits change.
Turn off track changes — the review round is done.
Track Changes and AI edits. When Track Changes is on, every edit WorClaw makes is recorded as a tracked change attributed to the current user. This is a great way to review AI edits before accepting them — let WorClaw restructure, then step through the tracked changes to accept or reject each one.

8.2 export_pdf

Exports the active document to a PDF file at a given absolute path. Overwrites any existing file at that path.

Export this document to PDF at C:\Users\me\Desktop\BrightRoute-Q2.pdf.

WorClaw uses Word's built-in export, so:

  • Track Changes marks are rendered in the PDF only if the document is set to show them (the same way Word itself prints).
  • Embedded fonts follow the document's save-time fonts settings.
  • Hyperlinks, bookmarks and the navigation pane are preserved.
Overwrites without asking. If the target path already exists, export_pdf will replace it. Pick a fresh file name if you want to keep previous exports.

8.3 Example recipes

Review round

Turn on track changes. Then tighten every paragraph in the "Key risks" section to under 40 words, keeping the meaning. Then list the paragraph indices you edited.

Final export

Turn off track changes. Save. Export to PDF at C:\Users\me\Documents\BrightRoute-Q2-final.pdf.

Versioned exports

Export this document to PDF at C:\Users\me\Desktop\v1.pdf. Then change every "draft" to "final" and export again to v2.pdf.

Chapter 9 — Settings

Chapter 9Settings

WorClaw's settings window is reached by clicking the gear icon in the chat panel header.

9.1 API key

Your Anthropic API key. Stored encrypted with Windows DPAPI under %APPDATA%\WorClaw\. The key is redacted in the field — click Show to reveal it temporarily.

Rotate your key if you share a machine. DPAPI protects keys from other Windows accounts on the same machine, but if you allow another person to use your Windows session, they can use your API key. Generate a new key in the Anthropic console and paste it here.

9.2 Model

Chooses which Claude model answers your messages.

ModelWhen to use
HaikuFast and cheap. Good for routine tasks with small inputs and short replies.
SonnetBalanced default. Handles summarisation, multi-step reasoning and longer drafts.
OpusHighest quality, slower and more expensive. Use for nuanced tone, sensitive material or hard judgement calls.

9.3 Confirmation policy

Controls how WorClaw treats write operations.

  • Always ask (default) — every write operation requires confirmation.
  • Ask for destructive only — inserts and style changes run without a prompt. delete_selection and export_pdf (which overwrites a file) still confirm.
  • Never ask — not recommended for shared or production documents.
Delete always confirms. Regardless of this setting, delete_selection is always gated behind an explicit yes / no prompt. This is hard-coded and cannot be disabled.

9.4 Logging

Controls the local log file used for diagnostics.

  • Off — no log written.
  • Errors only (default) — only failures are recorded.
  • Verbose — every tool call, input and result is written to the log.

Log file location: %APPDATA%\WorClaw\worclaw.log. It is a plain text file you can open in Notepad. Document, mailbox or workbook contents are not logged at any verbosity level.

9.5 Panel position

By default the chat panel docks on the right of the main Word window. You can float it or dock it on the left. The position is remembered per Windows profile.

Chapter 10 — Tips & troubleshooting

Chapter 10Tips & troubleshooting

10.1 Prompt patterns that work

  • Anchor on indices or headings. "After paragraph 7" or "under the 'Key risks' heading" is more reliable than "near the middle".
  • List before you edit. Ask WorClaw to list_headings or list_paragraphs before a complex restructure — it lets both of you agree on which paragraph is which.
  • Find before you replace. Run find_text first when you are not sure of the scope. A read costs pennies; a bad replace costs minutes of cleanup.
  • Use Track Changes for audits. Turn on Track Changes, let WorClaw edit, then step through the tracked changes. Safer than blind accept.

10.2 Common pitfalls

SymptomCauseFix
Style not appliedStyle name does not exactly match the document's style set.Open the Styles pane in Word to check the exact name. Pass it to WorClaw with the same spelling and capitalisation.
Heading list looks wrongDocument uses built-in headings in some places and custom styles in others.list_headings only returns Heading 1–4. If you use a custom heading style, re-apply built-in headings first.

10.3 Cost and token tips

  • Listing and reading are cheap. Drafting long sections is expensive. Ask for the cheapest step first.
  • Haiku is often good enough for structural work (inserts, style changes, replace). Switch to Sonnet for drafting and Opus only when tone really matters.
  • Clearing the conversation periodically resets the context window and reduces token usage on subsequent turns.

10.4 Bug-report helpers

Two built-in tools make it easy to gather the build fingerprint and capability surface when you need to report an issue:

  • get_version_info — ask "what version of WorClaw am I running?" and WorClaw replies with add-in name, auto-refreshed build version, the Office host version, OS, and .NET runtime. The same version number is always visible in the top-right corner of the Settings dialog, so you can grab it without sending a chat turn.
  • list_tools — ask "which WorClaw tools mention 'bibliography'?" and it filters the full tool catalogue by any substring of the name or description. Useful when you're not sure whether a capability exists, or when you want to compare what's available against the online docs.

10.5 Where to get help

  • Log file — %APPDATA%\WorClaw\worclaw.log. Set logging to Verbose before reproducing the issue.
  • kalmstrom.com support — send the log excerpt (with your API key redacted) and a description of what you asked and what happened.
  • Anthropic status — if WorClaw replies with "Claude is unreachable", check status.anthropic.com before worrying about your install.
The chat panel is a client. WorClaw does not have a backend of its own. Every reply comes straight from Anthropic, and every action happens in Word's process on your machine. Your data never visits kalmstrom.com.
